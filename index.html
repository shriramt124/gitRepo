Version git control

Git add * it means add all the files in the current directory to the staging area

Git commit -m "message" --> captures the snapshop of the project's currently staged changes ,creating a permanent record in the repository's history

git Log --->

git Log --oneline ---> is used to see the commit history of the repository in one line

git Log -p ---> is used to see the commit history of the repository with the changes made in each commit

git Log --stat ---> is used to see the commit history of the repository with the number of lines added or deleted in each commit

git Log --graph ---> is used to see the commit history of the repository in a graphical format

git Log --all ---> is used to see the commit history of the repository including all branches

git Log --author="authorname" ---> is used to see the commit history of the repository made by a specific author

git Log --before="date" ---> is used to see the commit history of the repository before a specific date

git Log --after="date" ---> is used to see the commit history of the repository after a specific date

git rm --cached filename --> removes the file from the staging area

git diff --> it is used to see the difference betweent the previous and curr of what you have changes

if we added the file in staging area and then we made some mistake to the file then to restore that changes after the staging area then we will use --> git restore --worktree index.html
